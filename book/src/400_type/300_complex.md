# Complex

## Strings

Strings are a complex type that are made of array of chars with null terminator '\0', and by default is utf8 encoded:
```
str[]
```

## Number
Number type is an abstraction of integer and float type in the current hardened
front-end. Imaginary-number support is still outside the active stream/lexer/parser
contract.
```
num[]
```

## Pointer
```
ptr[]
```
## Error
```
err[]
```
